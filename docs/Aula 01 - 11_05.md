## O que é requisito de usuário?
Declarações, em formato ou linguagem inteligivel pelo cliente, sobre as funções que o sistema deve oferecer e as restrições sob as quais deve operar
## O que é requisito de sistema?
Estabelecem detalhadamente as funções e as restrições do sistema, em um formato mais apropriado para a implementação
***
### Por que precisamos de requisitos?
* Para entender o que o cliente quer e entender o problema do negócio
* Para documentar o que o cliente quer e documentar o escopo do projeto e definir suas restrições
* Para assegurar a qualidade e a satisfação do cliente e definir critérios de aceitação e gerenciar as expectativas do cliente

### Requisitos funcionais
* Requisitos diretamente ligados ao comportamento do software
* Descrevem as funções que o software deve executar
* Um requisito funcional descreve uma interação entre o sistema e seu ambiente
### Requisitos não funcionais
* São requisitos que expressam condições que o software deve atender ou qualidades específicas que o software deve ter
* Em vez de informar o que o sistema fará, os requisitos não funcionais impõem restrições ao sistema
* Podem ser mais críticos que requisitos funcionais, chegando a tornar um sistema impossível ou inútil
	* Se o sistema não os satisfaz, muitas vezes o sistema se torna inútil

Requisitos não-funcionais podem ser mais críticos que os requisitos funcionais
